二. ArrayList构造方法解析
1.ArrayList()
此构造方法很简单，直接将elementData置为{},也就是一个空数组.

2.ArrayList(int initalCapacity)
此构造方法将判断initalCapacity是否合法，如果不合法(小于0)，则抛出非法异常。
合法则正常构建数组：
当initalCapacity==0 , elementData = { }
当initalCapacity>0 , elementData = new Object[initalCapacity]

3.ArrayList(Collection c)
此构造方法会将集合c中的元素复制一份存入elementData中。
逻辑如下：
首先将c转化为数组（调用toArray方法）并赋给elementData
然后判断c的大小是否不等于0，如果不等于0，则将其大小值赋给size，然后调用Arrays.copyOf方法重新开辟相同大小的Object数组空间，将elementData中的值复制到新Object数组中，并赋给elementData.
如果等于0，则还是elementData = { }

三.ArrayList常用方法解析
1.add(Object obj)方法解析
每次进入add方法，都会为修改记录变量modCount增1，然后判断elementData的长度是否等于size（满了）
如果满了则需要进行扩容，扩容原则如下：
如果使用的是默认无参构造方法，size1.5<=size+1，则为elementData开辟10个对象空间。
如果使用的是指定数量的构造方法，size1.5<=size+1，则为elementData开辟size+1个对象空间。
如果在原数组的基础上增加了一半后大于size+1且小于MAX_ARRAY_SIZE,则将数组扩容至150%大小。
然后将就原数组里的元素移到新的大数组中，最后进行赋值操作，size加一。

2.get(int index)方法解析
get方法根据index,得到elementData指定位置上的元素。
进入get方法，首先检查index, 如果index<0或者index>=size,则抛出数组越界异常。
如果正常，则返回elementData[index]元素的引用。

3.remove(int index)方法解析
进入此方法首先进行index的合法性检查，检查没问题进入下一步，否则抛出异常。
然后把elementData的引用赋给临时不可变变量es上，接着取出index位置上的元素赋给oldValue
接着进入删除操作中，首先将修改次数modCount加一，然后声明一个newSize = size-1
如果要删除的位置小于size-1位置，则进行下面操作：将i+1到size-1的一组数移到i到size-2上，否则啥都不做
接着es[newSize] = null , size = newSize，这样就完成了删除指定位置的元素的操作。

4.add(int index,Object obj)
首先对index进行合法性检查。
然后对修改记录数modCount进行加一操作。
然后判断当前elementData的size是否等于它的长度，如果等于，则进行扩容。
接着把index到size-1位置上的数往后移一位，紧接着 elementData[index] = obj
最后将size加一。
